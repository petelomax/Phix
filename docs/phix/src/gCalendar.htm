gCalendar

Definition: include theGUI.e

gdx id = gCalendar([rtn value_changed=NULL,] [object ymd=NULL,] string attributes="", dword_seq args={})
Description: As used on the internal drop-down of a gDatePick(), but made available for permanent display.
Note this is a fixed-size/as-is control and unlikely to work properly with different sizes/fonts [TO-DO].

The parameters (including paranormalisation), attributes, and handers are identical to gDatePick().
pwa/p2js: Supported.
Example: 
-- demo\theGUI\gCalendar.exw
include theGUI.e

gdx calendar = gCalendar(),
    hbox = gHbox({calendar}),`SPACE=AROUND`)
    dlg = gDialog(hbox,`Calendar`)
gMainLoop(dlg)
Notes: Clicking on the central header "scrolls up" to months, years, decades, and centuries (see technicalia), unless the specified RANGE is already fully on-screen - should, for instance, you have to book a visit within the next ten days, half the time you won't be able to even get off the initial single-month view, or scroll left/right, unless that ten-day window happens to straddle a month end.

Anything outside the specified RANGE should appear greyed-out, have no hover effects, and be non-clickable.

You can also use the mousewheel instead of the left/right buttons, when enabled, and of course if the mousewheel doesn't do anything (=== left/right arrows greyed out), neither will clicking on the header.
Technicalia See demo\theGUI\hCalendar.exw and hText.exw for progress on a canvas-based drop-in replacement.
The following are images of the native controls, which gCalendar/gDatePick replicate as best they can.


At the top is a Windows SysDateTimePick32 control with the calendar drop-down in four title-click-states (½|¾).
In fact, since it was trivial, I added a fifth for a thousand years, much like the fourth's century but 17% more 0s, plus the temptation to merge similar 3-way is stronger than 2-way, so the code base actually shrank.
Below it is a GtkCalendar, there being no such thing as a GtkDatePick, and which doesn't do any of that state-stuff.
I couldn't convince myself a today button was all that useful, so left that out, otherwise theGUI mimics Windows.

There has been no attempt to implement any smooth transitions, that is left as an exercise for the reader. It would probably involve three gImages: old, new, and mix, on which to paint the fade/slide/zoom effects, plus a timer.

The dropdown itself is just a headerless gCanvas window, to the below/right unless that would go off-screen, with appropriate KEY/CLICK/MOUSEWHEEL handing.

Before the switch to emulated controls, this needed manual re-implementation anyway, since HTML has a perfectly decent drop-down but no always-visible equivalent I can find, and GTK the opposite [whereas WinAPI has both].

Ideally I suppose prior month enabled, prior month disabled, and this month disabled should be displayed in three different shades of grey, but that would be so easily-missable it is not worth bothering, whereas the presence/lack of hover-effects is/should be obvious.

One of the trickiest parts to implement was in fact preventing the user from navigating wildly off-range, so by all means be suspicious, but please think carefully (re click-ability-ness) before you open your gob.
