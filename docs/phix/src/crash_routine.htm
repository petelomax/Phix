crash_routine

Definition: crash_routine(integer rid)
Description: Specify a function to be called when an error takes place at run time.

rid: a valid result from routine_id(), or (better yet) originating from some plain function identifier.
pwa/p2js: Not (yet) supported.
Comments: The supplied function should accept a single parameter, which is currently always 0.

Invoking crash_routine() adds a new function to a list of routines to execute upon a crash.
The routines defined first are executed last.

If a crash routine returns anything other than 0, the remaining routines in the chain are skipped.

Note that, unlike Euphoria, passing a rid of -1 clears the list.
Be aware that crash_routine(routine_id("mis_spelt_name")) has the same effect,
whereas crash_routine(mis_spelt_name) would more helpfully trigger a compilation error.

crash routines are not full fledged exception handlers (unlike try/catch), and they cannot resume execution.
However, they can otherwise perform almost any action, including, at least on desktop/Phix, reading the generated crash file, and perhaps emailing a support message and/or restarting the program, or in a web browser performing something a little more interactive than the standard JavaScript practice of logging something to a hidden console and then quietly soldiering on, and that way at least letting the user know something hadn't quite panned out.
Example: 
function report_error(integer /*dummy*/)
    mylib:email("maintainer@remote_site.org", "ex.err")
    return 0
end function
crash_routine(report_error) -- [same as crash_routine(routine_id("report_error"))]
Implementation: via :%pCrashRtn in builtins\VM\pDiag.e (an autoinclude) - be warned however it is low-level complicated stuff that you do not need to know.
See Also: try/catch, abort, crash_file, debugging and profiling
