Platform specific

Definition: include theGUI.e

-- then --

[requires(LINUX) -- optional but often sensible]
include theGUI.LNX.e -- typically GTK or syscall-based
    <Linux-specific program>

-- and/or --

[requires(WINDOWS) -- optional but often sensible]
include theGUI.WIN.e -- typically WinAPI-based
    <Windows-specific program>

-- and/or --

[requires(JS) -- optional but often sensible]
include theGUI.JS.e -- typically DOM-based
    <JavaScript-specific program>
Description: Hoepfully temporary storage space for [experimental] code which has not yet been made properly cross-platform.
Instead of a requires() statement it should be possible to use if platform()=LINUX|WINDOWS|JS then guards around any calls, and enable/disable or show/hide various buttons and menu entries on a similar basis.
Conversely a requires() statement is the best fit when there is little hope of some feature ever becoming cross-platform.
Some of the files above may be empty, and note theGUI.JS.e contains just enough code to compile cleanly on desktop/Phix along with crash statements to help identify where any guards have been forgotten, with pwa\theGUI.JS.js containing the real hand-crafted code.

Only once some feature has suitable alternatives in all three can they possibly be merged properly into theGUI itself.
There shouldn't be anything to stop you including two out of three, or indeed all three, possibly for different reasons.

The following lists all known platform/backend differences in theGUI.e/js:
  • The MNZ attribute is inoperative under GTK.
Potential candidates include loading/saving images, reading the Windows registry, downloading/uploading files, coping with some of the more exotic clipboard formats, etc. I plan to add some notes here once any such actually exists.
pwa/p2js: Supported, or at least whatever is in pwa\theGUI.JS.js is.
