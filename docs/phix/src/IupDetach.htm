IupDetach

Syntax: include pGUI.e

IupDetach(Ihandle child)
Description: Detaches an interface element from its parent.

child: Identifier of the interface element to be detached.
Notes: It will automatically call IupUnmap to unmap the element if necessary, and then detach the element.

If left detached it is still necessary to call IupDestroy to destroy the IUP element.

The elements are NOT immediately repositioned. Call IupRefresh for the container (or any other element in the dialog) to update the dialog layout.

When the element is mapped some attributes are stored only in the native system.
If the element is unmaped those attributes are lost.
Use the function IupSaveClassAttributes when you want to unmap the element and keep its attributes.
See Also: IupAppend, IupInsert, IupRefresh, IupUnmap, IupCreate, IupDestroy
