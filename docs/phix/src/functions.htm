Functions

These are just like procedures, but they return a value, and can be used in an expression, e.g. 
    function getAverage(atom a, atom b)
        return (a+b)/2
    end function
Any Phix object can be returned. You can return multiple values, by returning a sequence of objects. e.g. 
     return {x_pos, y_pos}
Generally function names should begin with a verb, mainly since eg atom average = average(...) is patently nonsense.

When a local (routine-scoped) variable is both passed to and assigned the result of a function call, phix applies automatic pbr (pass by reference) handling, which makes said unassigned over the call and hence may permit the function to modify it in-situ, which can significantly improve performance, for example: 
    procedure myproc()
        sequence table = tagset(100) -- nb must be local
        table = sq_mul(table,2) -- automatic pbr applies
    end procedure
Phix is a reference counted language and technically does not employ pbr, however whenever it manages to keep/pass the reference count as 1, the performance profile is identical. A similar situation applies to inline function calls used as parameters to another routine.

See also: Multiple Assignment

The general term "subroutine", or simply "routine" is used when a remark is applicable to both procedures and functions.
