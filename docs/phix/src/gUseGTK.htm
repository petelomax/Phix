gUseGTK

Definition: include theGUI.e
gUseGTK()
Description: Optional, Windows only, not formally supported, for quickly testing GTK on Windows.
pwa/p2js: Ignored, as per notes below.
Comments: This procedure should be called immediately after the include statement and before creating any controls, etc.
In fact, a fatal error occurs should this be invoked after the internal tg_Init() routine has already been called.

On Windows (only) it permits the GTK-based code to be tested, using some pre-installed dlls I found lying around.

On Linux it simply sets a flag which is already true to true, and is therefore effectively ignored, hence theGUI.js aka
using JavaScript in a browser now also has a similar "do nothing" routine.
See Also: gVersion, specifically with a bBack of -1.
Technicalia I must have tried at least 15 different sets of prebuilt GTK dlls, but only two actually worked for me.

My demo\theGUI\win_gtk32\ contains 2.24.10 (32bit) dated 08/02/2012 which I believe came from here.
There is also a slightly older 64 bit version (2.22, 29/12/2010) here but I don't think I ever tried it.

My demo\theGUI\win_gtk64\ contains 3.22.30 (64bit) dated 12/06/2018 which I believe came from here.

Note gUseGTK() won't work at all out-of-the-box, on Windows, unless at least one of those has been set up by hand.

I see little point adding 110MB of low-quality outdated files to the standard distro, you may be pleased to hear.

I never got GTK4 to work acceptably, on any platform, see here.
