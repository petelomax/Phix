gmg

Definition: include theGUI.e

gmg identifier [= gImage(...)]
Description: Type. A platform-specific image. May not be NULL.

Mainly intended for parameters to routines defined in theGUI itself, and/or to clarify the docs.
pwa/p2js: Supported/ignored as per gdx.
See Also: gdx
Technicalia A sequence such as {"gImage",{GtkPixbuf,w,h},iid} under GTK or {"gImage",{DIBitmap,w,h,t},iid} under WinAPI (at least as at the time of writing), or a HtmlElement under p2js, but all that is an implementation detail, subject to change in future releases, and an application should treat all such as opaque. While you should get away with printing such when debugging on desktop/Phix, printing under p2js is formally undefined behaviour or deliberately crashing.

An application can freely use gmg to define variables and clarify code, but it should not rely on implicit side-effects (other than sanity checks/helpful crashes) or explicitly use a type test for control flow purposes.
