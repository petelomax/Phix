match_any

Definition: integer res = match_any(sequence needles, haystack, integer start=1, mult=1)
Description: Find the first positon where any of the needles matches a slice of haystack.

needles: the list of items to look for
haystack: a sequence in which "needles" are looked for
start: the starting point of the search. Defaults to 1.
mult: if >1 (powers of 10 or 2 recommended) returns haystack_idx*mult+needle_idx else just the haystack_idx.

Returns 0 if none found, or the smallest start of a slice in haystack matching some element of needles, adjusted by mult>1 as above. An error occurs if mult!=1 but not >length(needles).

Note it actually invokes begins() at each position in haystack for each element in needles, in order to yield the first, and may therefore prove rather costly, and perhaps especially so should you not be using start to cut out completely unnecessary work on long haystacks. For instance, were you replacing all double-vowels in a long string with one, it would be pretty daft after replacing s[9002..3] to look for the next pair from [1]. It may also be significantly more efficient to use substitute_all() or some variant of the code from substitute() based on a match_all() result instead - but of course this is perfectly fine for reasonably short strings/single lines.
pwa/p2js: Supported.
Example: 
location = match_any({"o"}, "John Doe")             -- location is 2
location = match_any({"o","i"}, "John Smith", 3)    -- location is 8
Implementation: See builtins\pfindany.e (an autoinclude) for details of the actual implementation.
