deprecated

The arwen gui library is windows 32-bit only, and hence it is recommended that any new programs use theGUI, which is 32/64 bit and Windows/Linux compatible. It is however still supported and may be appropriate for quick one-offs.

There are no plans to remove, drop support, or deliberately break anything in this section, however there will be few if any further enhancements to them.

Only a few of these routines are supported by pwa/p2js: beep, bigatom (which has the advantage over mpfr that no dlls are required, not applicable under p2js anyway, though it is exercised in several rosettacode entries - Catalan_numbers[_Pascals_triangle], fibonnaci, and Miller_Rabin_primality_test), and pGUI (still usable but superceded by theGUI, which has the advantage that IUP does not need distributing/installing, though the Windows dlls will be removed in some 1.0.6+ version and after whenever that happens need downloading separately).

Arwen - a fast, compact GUI library for Windows-based applications (32-bit only).
arwen32dib - fast bitmap manipulation (24-bit graphics).
beep - a (somewhat painfully) simple beep procedure (console or gui).
bigatom - arbitrary precision arithmetic.
console graphics - from/compatible with Euphoria's DOS beginnings.
database - simple database system (copied from Euphoria).
librsvg - [draft] svg (scaleable vector graphics) handling.
message_box - windows only, so (eg) gMsgBox() would be better.
nopoll - incomplete (was at one point considered as a possible alternative to libcurl).
pGUI - The Phix portable Graphical User Interface - 32/64 bit, Windows/Linux/p2js compatible, IUP based.
read_bitmap - does not and never did support 24 or 32 bit colours.

Technically, demo\Win32Demo/lib6|7 are also in this category, but I've never documented them.
