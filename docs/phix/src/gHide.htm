gHide

Definition: include theGUI.e

gHide(gdx id)
Description: Hides an interface element, which can be anything from an entire gDialog down to just a single field.
id: Identifier of the interface element.

Equivalent to gSetAttribute(id,`VISIBLE`,false) when used on individual elements and sub-containers, however when that is used on a dialog it is hidden but the message loop is not [currrently] terminated, unlike gHide().
pwa/p2js: Supported. In some cases, closing the last dialog automatically closes the browser tab, but that is in no way assured, and may in fact be expressly prohibited in brower settings, and sometimes even by a popup blocker (ones that don't understand the word "up", I assume).
Notes: When a dialog is hidden, by any of (at least) gHide(), setting VISIBLE to false, clicking on the window close button, or returning TG_CLOSE, the elements inside the dialog are not destroyed, and can be shown again. Hiding the last visible dialog terminates the event loop, but only if no others were hidden by setting their VISIBLE attribute, hence the latter is in general strongly discouraged, on whole dialogs that is, at least unless very carefully managed/restored.
See Also: gShow, VISIBLE
