gProgressBar

Definition: include theGUI.e

gdx id = gProgressBar(string attributes="", dword_seq args={})
Description: Creates a progress bar control. Shows a percent value that can be updated to simulate a progression.

For more information on the attributes and args parameters see gSetAttributes().

Returns: the identifier of the created element.
pwa/p2js: Supported.
Example: 
-- demo\theGUI\gProgress.exw
include theGUI.e
gdx pbar
atom f = 0

procedure action(gdx timer)
    f += 0.1
    gSetAttribute(pbar,"VALUE",f)
    if f>=1 then
        gSetAttribute(timer,"RUN",false)
    end if
end procedure

pbar = gProgressBar()
gdx hbox = gHbox({pbar},"MARGIN=9x9"),
    dlg = gDialog(hbox,"gProgressBar",`SIZE=240x80`),
    timer = gTimer(action,1000,true)
gMainLoop(dlg)
Notes: This is implemented using the native controls, with all the quirks, limitations, weirdness, hiccups, and gotchas that entails. It should not be even the slightest bit difficult to create a gCanvas() of the desired size and draw exactly as wanted, that is, should this control not meet your needs or expectations. For instance I would struggle to convince the native controls to display "20%", let alone consistently between backends and at which of the 40-ish places I can immediately think of (l/m/r * in/out * t/m/b * in/out + moving). Admittedly Windows has a distinctive rolling gradient that might be fairly hard to replicate, but I'm not really convinced I like that anyway. Obviuously there is nothing to stop you putting such text next to a progress bar, rather than inside it, to keep things simple.

Some examples can be seen in demo\theGUI\sample.exw
Attributes:
MIN,
MAX 		(non inheritable) Contain the minimum and maximum value. Defaults are 0 and 1.
The control display is not updated, must set the VALUE attribute to update.
There are also MNZ and RST attributes on a gDialog, and the min() and max() builtins, were you looking for them.
ORIENTATION (creation only) can be "VERTICAL" or "HORIZONTAL". Default: "HORIZONTAL".
Horizontal goes from left to right, and vertical from bottom to top.

Also present on gBox and gSlider, in case you were looking for those.
SIZE The initial size is defined as "200x30", or "30x200" if vertical.
VALUE (non inheritable) Contains a number between "MIN" and "MAX", controlling the current position.
also ACTIVE, EXPAND, FONT, MARGIN, MINSIZE, MAXSIZE, TIP, VISIBLE.
