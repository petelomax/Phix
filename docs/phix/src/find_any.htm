find_any

Definition: integer res = find_any(sequence needles, haystack, integer start=1, mult=1)
Description: Find any element from a list inside a sequence. Returns the location of the first hit.

needles: the list of items to look for
haystack: a sequence in which "needles" are looked for
start: the starting point of the search. Defaults to 1.
mult: if >1 (powers of 10 or 2 recommended) returns haystack_idx*mult+needle_idx else just the haystack_idx.

Returns 0 if no needle is found, or the smallest index in haystack of an element of needles, adjusted by mult>1 as above. An error occurs if mult!=1 but not >length(needles).

Note it actually looks for elements of haystack in needles, in order to yield the first position, and may therefore prove rather costly should you not be using start to cut out completely unnecessary work on long haystacks. For instance, were you replacing all vowels in a long string with spaces, it would be pretty daft after replacing s[9002] to look for the next vowel from [1].
pwa/p2js: Supported.
Example: 
location = find_any("aeiou", "John Doe")        -- location is 2
location = find_any("aeiou", "John Smith", 3)   -- location is 8
Implementation: See builtins\pfindany.e (an autoinclude) for details of the actual implementation.
